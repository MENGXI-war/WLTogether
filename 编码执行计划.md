### 1.3 认证模块（0.5 天）

| 文件 | 内容 |
|------|------|
| `config/SecurityConfig.java` | Spring Security 配置：放行 /api/auth/* 和 /api/server/capabilities，其余需 JWT |
| `security/JwtTokenProvider.java` | 生成/验证 accessToken(2h) + refreshToken(30d)，载荷 {sub, username, role, iat, exp} |
| `security/JwtAuthFilter.java` | 从 Authorization Header 提取 Bearer token，设 SecurityContext |
| `security/RefreshTokenBlacklist.java` | 内存 ConcurrentHashMap，轮换时旧 token 入黑名单 |
| `model/dto/LoginRequest.java` | {username, password} |

| `controller/AuthController.java` | POST /api/auth/register、/api/auth/login、/api/auth/refresh |
| `service/AuthService.java` | 注册(BCrypt哈希) / 登录(校验+生成JWT) / refresh(校验旧refreshToken+轮换) |

**测试验证：**
- `POST /api/auth/register` → 201
- `POST /api/auth/login` → 200 {accessToken, refreshToken, user}
- `GET /api/users/me`（不带 token） → 401
- `GET /api/users/me`（带 token） → 200
- `POST /api/auth/refresh` → 200 新 token

## 阶段二：实时通信（2–3 天）

> 目标：WebSocket 连接建立，聊天实时收发，播放同步指令可广播

| `websocket/WsAuthInterceptor.java` | 从 ?token={jwt} 校验，将 userId 存入 session attributes |

### 2.2 聊天消息（0.5 天）

| 文件 | 内容 |

| `controller/ChatController.java` | GET /api/groups/{id}/messages（游标翻页） |
| `service/ChatService.java` | 消息持久化、分页查询、未读计数更新 |

### 2.3 播放同步（1 天）

| 文件 | 内容 |
|------|------|
| `websocket/SyncHandler.java` | 处理 sync.ping → 记录 t2 → 返回 sync.pong{t1, t2} |

| `service/PlaybackSyncService.java` | 播放模式校验（HOST_ONLY/FREE/SHARED_CONTROL）、共享控制防抖(3秒1次seek)、切歌缓冲窗口(2秒) |

### 2.4 会话生命周期 + P2P 信令（0.5 天）

| 文件 | 内容 |
|------|------|
| `websocket/SessionLifecycleHandler.java` | 处理 session.start/join/leave/end → 广播对应通知 |

| 12 | WiFi 断开 → 传输暂停 → 恢复 → 续传 | 不丢数据 |
| 13 | 429 限流 → 客户端退避 | 指数退避生效 |
| 14 | WS 断开 → 自动重连 | 退避 2s→5s→10s→30s |

         ┌── 阶段一(后端核心) ──┐

                    │

  │ Web 前端     │ │ Android    │
  │ 5-7 天       │ │ 5-7 天     │  ← 可并行
  └──────┬──────┘ └─┬──────────┘
         │          │
         └────┬─────┘
              │

    │ 阶段五(后台管理)     │
